Coverage Map.

Every brief answers a question for one family. This is the heatmap of where the catalog is deep and where it's thin — so an agent picking an audit, or a human proposing a new brief, can see the whole surface at once. Hotter means more briefs. It reads the same coverage.json that agents fetch.

Coverage by family

Each tile is one family, shaded by how many briefs audit it and sorted densest first — so the cool tail at the end is exactly where coverage thins out. Select a tile to list its briefs.
