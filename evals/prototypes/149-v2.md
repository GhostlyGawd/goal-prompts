# Goal: The Charter (decide class — contract v2 prototype)

You are working inside this repo. Mission: excavate what this project is
actually for — from the README's promises, the docs, and the change
history — and draft the one-page contract that keeps later work honest.

Inspect files and history with the capabilities your environment provides.
Internal subagents are fine; the user sees only this conversation.

## 1 — Inspect
Read the README, docs, and configuration; scan the change history for what
the project keeps investing in versus what it abandoned.

## 2 — Evaluate
Reconstruct: the problem, the intended user, the job to be done, apparent
non-goals, and invariants the code already defends. Note where evidence
conflicts and where you are guessing.

## 3 — Interview before drafting
The seven charter decisions (problem, user, job, non-goals, invariants,
now/next/not-now, done-looks-like) belong to the operator. Present each as
a numbered question WITH the evidence-backed candidate answers you found.
**If the operator is present, stop here and wait for answers. If this is an
unattended run, do not invent answers:** write the question sheet itself as
a DRAFT charter and stop.

## 4 — Save the result
Write `CHARTER.md` at the repo root — or in `reports/` if that directory
already exists and the root is reserved. Date it. Mark every unanswered
question **open**. The document is the artifact; there is no code to
change. If a previous CHARTER.md exists, open with what changed since.

## 5 — Continue in this conversation
Walk the operator through the sharpest tension you found (evidence, why it
matters, what ratifying each candidate would mean). If they answered the
questions, fold the answers in and ask them to ratify; if not, end by
asking which question to settle first. No edits beyond the charter file
itself; publishing or announcing anything needs its own authority.

## Rules
- Evidence or it stays an open question — never silently fill a decision.
- Plain words; no internal taxonomy in the document.
- Stay portable: no host-specific commands.
