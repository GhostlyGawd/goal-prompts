# CLAUDE.md ‚Äî working in the goal-prompts repo

This repo is a catalog of audit briefs for coding agents. It builds a static
site and machine-readable surfaces from markdown sources. No framework, no
runtime backend; the build is stdlib-only Python.

## Commands
- `python3 build.py` ‚Äî lint every brief and regenerate all outputs
- `scripts/check` ‚Äî the full gate: build + JS parse + MCP smoke + linter tests
- `python3 tests/test_build.py` ‚Äî linter unit tests

Run `scripts/check` before committing. CI runs the same.

## Layout
Edit these (source of truth):
- `prompts/<family>/NN-slug.md` ‚Äî the briefs (front matter + body)
- `playbooks.json` ‚Äî curated brief sequences
- `template.html` ‚Äî the catalog UI (build injects prompt + playbook JSON)
- `studio.html` ‚Äî Report Studio (standalone page)
- `build.py` ‚Äî the build + linter
- `mcp/server.cjs` ‚Äî the MCP server (zero deps)
- `install` ‚Äî the slash-command installer
- `js/` ‚Äî the site's pure-logic modules, each with a zero-dep Node suite:
  `catalog-core.js` (search/conductors ‚Äî `tests/catalog_core.test.cjs`),
  `report-parser.js` (Studio parsing ‚Äî `tests/report_parser.test.cjs`),
  `gp-detail.js` (detail-page context/copy helpers). `scripts/check` runs
  `node --check` on all of them plus both test suites, the hermetic
  installer test (`scripts/test-install.sh`), and the MCP smoke test
  (`scripts/mcp-smoke.cjs`, which also guards conductor-text parity
  between `build.py`, `mcp/server.cjs`, and `js/catalog-core.js`).

Never hand-edit these (regenerated by the build):
`index.html`, `raw/`, `b/` (per-brief detail pages), `p/` (per-playbook
detail pages), `plugin/` (the Claude Code plugin ‚Äî commands and
`plugin.json`, version from `package.json`), `catalog.json`,
`bodies.json` (every brief body in one blob ‚Äî the landing page fetches it
at copy/quick-view time and the SW precaches it; `raw/` stays the agents'
network-only endpoint so its fetch counts stay an honest usage metric),
`changelog.html` (the `/changelog` page, rendered from `CHANGELOG.md` ‚Äî
edit that), `checksums.txt`, `commands.tar.gz`, `commands.zip`,
`sitemap.xml`, `robots.txt`, `sw.js`, `tokens.css`.
(`og/p-<key>.png` playbook share cards are generated by `scripts/og.py
--playbooks`; the build fails if one is missing or its baked-in stage count
drifts from `playbooks.json` ‚Äî same guard as `og.png`'s brief count.)
(The service worker's body lives in `build.py` as `SERVICE_WORKER`; edit
it there. Its cache version is a content hash, so a deploy self-invalidates.)
(`studio.html` is a source page; Vercel's `cleanUrls` serves it at `/studio`.)
(`img/studio.png` is a real Report Studio screenshot regenerated with `node
scripts/studio-shot.cjs` ‚Äî needs the global `playwright` package + the pre-installed
Chromium. It's an ungated asset, so re-run it when the Studio UI changes.)
(Design tokens are one source of truth: `TOKENS_CSS` in `build.py` ‚Üí written to
`tokens.css`, which every surface links ‚Äî `template.html`, the `b/`/`p/` detail
pages, `studio.html`, `vitals.html`. Edit tokens there, never in the HTML.
`SITE_CSS` and the `brief_detail`/`playbook_detail` builders hold the detail
pages' component CSS; `template.html` holds the landing page's component CSS.)

## Conventions
- Every brief follows the 4-phase skeleton (Explore ‚Üí Audit ‚Üí Curate ‚Üí Report)
  and contains the ask-first gate ("Report only ‚Äî end by asking ‚Ä¶"). The
  linter enforces this on every brief.
- Audit briefs are read-only and write one report file. The one exception is
  47 The Fixer, which modifies code but gates on an explicit selection first ‚Äî
  its ask-first gate sits at the end of Phase 2, before any edit happens.
- Brief bodies must stay under 4,000 characters.
- Front matter may carry `related:` (a list of brief ids) ‚Äî rendered as
  "pairs well with" cards on the brief's `/b/<id>` page. The linter fails on
  unknown ids.
- Family order and questions have one source: `FAMILY_ORDER` in `build.py`
  plus the briefs' front matter, injected into the site as
  `__FAMILIES_JSON__`. Family *colors* also have one source now:
  `build.FAMILY_COLORS` ‚Äî `scripts/og.py` imports it, and `build.py` emits the
  `.f-*` rules into `tokens.css` from it (they used to be duplicated in
  `template.html`). Add a family in `FAMILY_ORDER` + `FAMILY_COLORS`; nothing
  else needs editing.
- Icons & marks: the UI icon language is deliberately a small Unicode set in
  the mono font ‚Äî `‚Üí` (leads a report filename), `‚ñ∏` (expander chevrons),
  `‚úì` (done/success), `√ó` (remove), `‚Üó` (external link). Don't add an icon
  library; if a new glyph is needed, pick one the mono stack renders and add it
  here. The brand mark is the 4-bar equalizer (`build.BRAND_MARK`); the favicon,
  `icons/*.png` (via `scripts/icons.py`), and each OG card's left bar all derive
  from it ‚Äî keep them in step.
- Playbooks in `playbooks.json` may carry optional merchandising fields ‚Äî
  `type` (standard | themed | collab | sponsored), `featured`, `badge`,
  `window`, `accent`, `partner`, `preview`, `tagline`. They drive the
  storefront on the landing page and each playbook's `/p/<key>` page; all
  are optional and default to a plain "standard" playbook.

## Node
Only `mcp/` and the JS syntax checks need Node. Brief and site work need only
Python and a browser.
