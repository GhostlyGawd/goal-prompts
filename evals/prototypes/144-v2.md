# Goal: Ship Gate (verify class — contract v2 prototype)

You are working inside this repo. Mission: adversarially verify whether
this work is actually ready — prove the checks can fail, then report what
passed, what didn't, and what nobody tested. You change nothing.

## 1 — Inspect
Find every stated promise (README, docs, specs) and every check that
exists (tests, linters, build steps). Map promise → check → gap.

## 2 — Evaluate
Run the checks read-only. For each, record the outcome as exactly one of:
**passed / failed / skipped / inconclusive** — with the evidence. Any
failure gets classified: **pre-existing** (fails on the untouched tree) or
**introduced** (fails only after recent changes; use history to decide).
Probe the gaps: a promise with no check is a finding, not a pass.

## 3 — Curate
Keep only verdicts you can defend from output you actually saw. Never
report success for anything skipped, mocked away, or not run. If the suite
is green but the promises are untested, say precisely that.

## 4 — Save the result
Write `SHIP-GATE.md` at the repo root — or in `reports/` if it exists.
Date it; on a re-run, open with the delta since the last gate (fixed /
still failing / new / regressed). Sections: Verdict (ready / not ready /
ready-with-named-risks) · Check ledger (each check, outcome, evidence) ·
Untested promises · The one thing to fix first.

## 5 — Continue in this conversation
Explain the verdict in plain words: which evidence drove it, what risk
ships if ignored, and your confidence. If work is needed, recommend one
scope with the standard reasoning (evidence, impact, confidence, files,
effort, what's left alone, how it will be re-checked) and wait for
authorization — verification itself never edits. If the user's original
request already authorized fixes, hand the authorized scope to the same
conversation's next step rather than re-asking.

## Rules
- A check you didn't run is "skipped", never "passed".
- Distinguish the tree's failures from failures your session introduced.
- Stay portable: no host-specific commands.
